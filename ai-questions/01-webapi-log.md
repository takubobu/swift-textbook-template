### Q1

**質問：**

Swiftのguard let url = URL(string: urlString) else { return }という書き方はどんな役割をしていますか？また、if letとの違いはどんなところですか？初学者向けに教えてください。

**AIの回答の要点：**

Swiftのguard letは、アプリがクラッシュしたり予期せぬ動作をしたりするのを防ぐための**「超強力な門番（ガードマン）」**のような役割をしています。

Swiftでの役割： 「安全確認」と「不純物の排除」

決定的な3つの違い
1. 有効範囲（スコープ）が違う
   if let: 取り出した変数は、if の { } の中でしか使えません。
  guard let: 取り出した変数は、guard を抜けた 下の行すべてで使えます。

2. 早期リターンが必須
   if let: 失敗しても、何もしないでスルーできます。
   guard let: else { ... } の中で必ず return や throw を書いて、処理を中断させなければいけないというルールがあります。

3. ネスト（入れ子）の深さが違う
   これが一番のメリットです。チェックしたい項目が3つ、4つと増えた場合を想像してください。
   if let の場合:「もしAなら…、もしBなら…、もしCなら…」と書くと、コードがどんどん右にズレていき、迷路のようになります（通称：ピラミッド・オブ・ドゥーム／破滅のピラミッド）。
   guard let の場合:「Aがダメなら帰れ」「Bがダメなら帰れ」「Cがダメなら帰れ」と、ダメなものを次々に弾いていくので、メインの処理を常に画面の左端（ネストなし）に書くことができます。
 

**自分の理解：**

if-letとguard-letの有効範囲の違いを勘違いしていたので今回学べてよかった。
プログラムを書いているときにif-letとごちゃごちゃにならないように気をつけつつ、違いを確認しながらプログラムを作成していきたい。

### Q2

**質問：**
addingPercentEncodingを使って検索テキストをエンコードしていますが、これをしないとどうなりますか？特に、日本語のアーティスト名(澤村光彩)で検索する場合に何が起きるのか教えてください。

**AIの回答の要点：**
結論から言うと、これを使わずに「澤村光彩」という日本語で検索しようとすると、アプリは通信を開始することさえできず、エラーで止まってしまいます。

1. URLには「半角英数字」しか使えないルールがある

   インターネット上の住所であるURLには、基本的に ASCII文字（半角の英数字や一部の記号） しか使えないという世界共通の厳しいルールがあります。
   「澤」や「光」といった日本語（全角文字）は、このルールに含まれていません。

2. 「澤村光彩」で検索した時に起きること

   エンコードをせずに日本語をURLに直接入れると、Swiftのプログラムは以下のように判断します。

   プログラムの視点: 「このURL、ルール違反の文字（日本語）が入っているよ！これじゃあ正しいURLとして組み立てられないから、処理を中断するね」

   コードの動き: URL(string: urlString) の部分で「正しいURLではない」と判定され、結果が nil（空っぽ）になります。

   結果: その後に続く guard let url = ... else { return } の else に引っかかり、検索処理そのものが実行されずに終わってしまいます。

**自分の理解：**

なぜエンコードが必要なのかをしっかり理解していなかったのでASCII文字にしないといけない理由を理解できた。
自分で制作するときにも使う可能性が高いのでそのときにはエンコードを忘れずにプログラムをしていきたいです。

### Q3

**質問：**
通信処理をdo-catchで囲っていますが、これはどんなことを想定して書かれているものですか？また、機内モードなどで通信ができないときに、このコードがどう機能するのか教えてください。

**AIの回答の要点：**
通信処理における do-catch は、アプリを**「不測の事態で落ちないように守るシールド」**のような役割をしています。

1.プログラミングには、自分のコードが正しくても防げない**「外部要因のエラー」**が必ず発生します。通信はその代表例です。

   1.サーバーのダウン: 相手のコンピューターが故障している。

   2.ネットワークの遮断: トンネルに入ったり、Wi-Fiが切れたりする。

   3.データの不整合: 期待していた音楽データではなく、エラーメッセージ（JSONではないもの）が返ってきた。

   これらの事態が起きたとき、try（やってみる）に失敗するとエラーが投げられます。もし do-catch がないと、アプリはそのエラーを処理できずに強制終了（クラッシュ）してしまいます。


2.機内モードやオフラインの状態でこのコードが実行されると、以下のようなステップで動きます。

   1.try await でエラー検知: URLSession が通信を試みますが、ネットワークが使えないため即座に「オフラインエラー」を発生させます。

   2.catch ブロックへジャンプ: 通信に失敗した瞬間、do の中のそれ以降の処理（データの解析など）はスキップされ、即座に catch の中身が実行されます。

   3.安全な事後処理: * print("エラー: \(error.localizedDescription)") により、コンソールに「インターネットに接続されていません」といった内容が表示されます。

   songs = [] が実行されることで、画面上のリストが空になり、アプリは**「壊れることなく、静かに検索結果がない状態」**を維持します。

   最後に isLoading = false が動くので、読み込み中のぐるぐる（ProgressView）も正しく消えます。

**自分の理解：**

通信エラーになったときの処理の手順を改めて知れたのと、どんなときに通信エラーになるのか、どんなことに気をつければいいのかも確認できた。
